# 如何创建云服务器实例

> **适用产品**：SmartCloud GPU 云主机 / ECS 弹性云服务器
> **最后更新**：2026 年 5 月
> **文档版本**：v2.1

> 适用角色：用户自助、客服、售前、技术支持
> 适用产品：云服务器 ECS、GPU 云主机、控制台创建流程
> 标签：创建实例、ECS、GPU、镜像、网络、安全组、购买流程

## 快速结论

- 创建实例前先确认业务类型、地域、实例规格、镜像、云盘和网络策略。
- AI 场景优先确认 GPU 规格和镜像兼容性，普通业务通常先从通用型实例起步。
- 如果面向中国大陆公众提供服务，还要提前关注备案和地域选择。
- 创建成功后不要只看实例“已开通”，还应继续验证登录、网络、云盘和 GPU 环境。

## 概述

本文档介绍如何在 SmartCloud 控制台上创建一台云服务器实例，涵盖从选型、配置、网络设定到最终开通的全流程。创建前请确保您已完成账号注册、实名认证，并了解所在区域的资源可用情况。

---

## 前提条件

| 序号 | 条件 | 说明 |
|:---:|------|------|
| 1 | 已注册 SmartCloud 账号 | 支持手机号或邮箱注册 |
| 2 | 已完成实名认证 | 个人认证或企业认证均可，ICP 备案要求企业认证 |
| 3 | 账户余额 ≥ 预估费用 | 按量付费需余额充足；包年包月需一次性支付 |
| 4 | 已了解目标区域库存 | GPU 实例部分区域库存紧张，建议提前查看 |

---

## 第一步：登录控制台

1. 打开浏览器，访问 **SmartCloud 控制台**：`https://console.smartcloud.cn`
2. 输入账号和密码，完成登录。
3. 在左侧导航栏选择 **云服务器 ECS** → **实例管理**。
4. 点击页面右上角 **创建实例** 按钮，进入创建向导。

> **提示**：首次使用建议切换至「新手引导」模式，系统会自动推荐常用配置。

---

## 第二步：选择地域与可用区

地域（Region）决定实例的物理数据中心位置，可用区（Zone）是同一地域下的电力和网络隔离区域。

| 地域 | 可用区 | 适用场景 |
|------|--------|----------|
| 华东-上海 | Zone A / Zone B | 国内业务首选，延迟低 |
| 华北-北京 | Zone A | 政企合规、ICP 备案 |
| 华南-广州 | Zone A | 南方用户、跨境出海 |
| 亚太-新加坡 | Zone A | 东南亚业务、免备案 |
| 欧洲-法兰克福 | Zone A | 欧洲合规、GDPR |

**选择原则**：

- **就近选择**：选择离您的最终用户最近的地域，降低访问延迟。
- **合规要求**：如需 ICP 备案，必须选择中国大陆地域。
- **跨区容灾**：生产环境建议在不同可用区各部署一台，避免单点故障。

> **注意**：实例创建后地域不可更改，请谨慎选择。

---

## 第三步：选择实例规格

### 3.1 通用计算型（ECS）

| 规格 | vCPU | 内存 | 网络带宽 | 适用场景 |
|------|:----:|:----:|:-------:|----------|
| ecs.t6.small | 1 | 2 GB | 1 Gbps | 开发测试、轻量 Web |
| ecs.c6.large | 2 | 4 GB | 2 Gbps | Web 应用、中小数据库 |
| ecs.c6.xlarge | 4 | 8 GB | 3 Gbps | 中型业务、微服务节点 |
| ecs.c6.2xlarge | 8 | 16 GB | 5 Gbps | 高并发服务、数据处理 |
| ecs.m6.4xlarge | 16 | 64 GB | 8 Gbps | 大型数据库、内存缓存 |

### 3.2 GPU 加速型

| 规格 | vCPU | 内存 | GPU | GPU 显存 | 适用场景 |
|------|:----:|:----:|:---:|:-------:|----------|
| gpu.gn6.small | 4 | 16 GB | 1× A10 | 24 GB | 推理服务、模型部署 |
| gpu.gn6.large | 8 | 32 GB | 1× A100 | 40 GB | 大模型训练、科学计算 |
| gpu.gn6.2xlarge | 16 | 64 GB | 2× A100 | 80 GB | 分布式训练、LLM 微调 |
| gpu.gn6.4xlarge | 32 | 128 GB | 4× A100 | 160 GB | 大规模并行计算 |

**选型建议**：

- **Web 服务 / 微服务**：ecs.c6.large 或 ecs.c6.xlarge
- **数据库**：ecs.m6.4xlarge（高内存型）
- **AI 推理部署**：gpu.gn6.small（性价比优）
- **大模型训练**：gpu.gn6.2xlarge 以上

---

## 第四步：选择镜像

镜像是实例的操作系统模板，包含 OS 和预装软件。

| 镜像类型 | 说明 | 推荐场景 |
|----------|------|----------|
| 公共镜像 | SmartCloud 官方维护，含 Ubuntu / CentOS / Debian / Windows | 通用部署 |
| GPU 镜像 | 预装 NVIDIA 驱动 + CUDA + cuDNN + Docker | AI/ML 工作负载 |
| 市场镜像 | 第三方提供的预配置环境（如 PyTorch、TensorFlow 一键部署） | 快速搭建框架环境 |
| 自定义镜像 | 从已有实例制作的快照镜像 | 批量复制相同环境 |

**操作步骤**：

1. 在「镜像」区域选择镜像类型标签。
2. 从列表中选择目标镜像，如 `Ubuntu 22.04 GPU`。
3. 确认镜像版本号和预装组件列表。

> **重要**：GPU 实例务必选择 GPU 镜像，否则需手动安装驱动，流程复杂且易出错。

---

## 第五步：配置存储

### 5.1 系统盘

| 类型 | 最大容量 | IOPS | 适用 |
|------|:-------:|:----:|------|
| ESSD PL0 | 500 GB | 10,000 | 通用、成本优先 |
| ESSD PL1 | 1 TB | 50,000 | 生产环境推荐 |
| ESSD PL2 | 2 TB | 100,000 | 高性能数据库 |

- 系统盘最小 **40 GB**，推荐 **100 GB** 以上（GPU 镜像需 ≥ 100 GB）。
- 系统盘不支持缩容，请按需规划。

### 5.2 数据盘

- 按需添加，最多挂载 **16 块**数据盘。
- 数据盘支持独立快照和扩容，与实例生命周期解耦。
- 建议训练数据、模型权重存放在数据盘，与系统盘隔离。

**操作步骤**：

1. 点击 **添加数据盘**。
2. 选择云盘类型、容量和加密方式。
3. （可选）勾选「随实例释放」，实例删除时自动销毁数据盘。

---

## 第六步：配置网络

### 6.1 专有网络 VPC

| 配置项 | 说明 |
|--------|------|
| VPC | 选择已有 VPC 或创建新 VPC |
| 交换机 | 选择 VPC 下的交换机，决定可用区和网段 |
| 公网 IP | 是否分配公网 IPv4 地址 |

### 6.2 带宽计费模式

| 模式 | 计费方式 | 适用场景 |
|------|----------|----------|
| 按固定带宽 | 按带宽峰值包月 | 流量稳定、长期运行 |
| 按使用流量 | 按实际流量 GB 计费 | 突发流量、短期任务 |

### 6.3 安全组

安全组是虚拟防火墙，控制入站/出站规则。

| 规则 | 协议 | 端口 | 来源 | 说明 |
|------|:----:|:----:|------|------|
| 入站 | TCP | 22 | 0.0.0.0/0 | SSH 登录（建议限制为办公网段） |
| 入站 | TCP | 80 | 0.0.0.0/0 | HTTP 服务 |
| 入站 | TCP | 443 | 0.0.0.0/0 | HTTPS 服务 |
| 入站 | TCP | 8888 | 10.0.0.0/8 | Jupyter Notebook（仅内网） |
| 出站 | ALL | ALL | 0.0.0.0/0 | 默认允许所有出站 |

> **安全提醒**：切勿将 SSH 22 端口暴露为 `0.0.0.0/0`，建议限制为办公 IP 段或通过 VPN/Bastion 访问。

---

## 第七步：系统配置

### 7.1 登录凭证

| 方式 | 说明 |
|------|------|
| 密钥对 | **推荐**。SSH 密钥登录，安全性高，支持免密登录 |
| 自定义密码 | 设置 root/用户密码，需满足复杂度要求 |

### 7.2 实例名称与标签

- 实例名称：便于管理识别，如 `web-prod-sh-01`
- 标签（Tag）：键值对，用于分类和成本归集，如 `Env: Production`、`Team: AI-Platform`

### 7.3 高级选项

| 选项 | 说明 |
|------|------|
| 自动恢复 | 实例故障时自动重启（默认开启） |
| 实例自定义数据 | 启动时执行 cloud-init 脚本，用于自动化初始化 |
| 专有宿主机 | 将实例调度到指定物理机，满足合规隔离要求 |

---

## 第八步：确认配置与支付

1. 在 **配置确认** 页面，逐项核对所有配置项。
2. 选择购买时长：
   - **按量付费**：按秒计费，随时释放，适合短期任务和开发测试。
   - **包年包月**：月付/年付享折扣，适合长期生产环境。
3. 阅读并勾选《云服务器 ECS 服务条款》。
4. 点击 **确认订单** → 完成支付。

> **预估费用参考**：
> - ecs.c6.large（按量）：约 ¥0.36/小时
> - gpu.gn6.small（按量）：约 ¥6.80/小时
> - 包年包月通常为按量价格的 6-7 折

## 第九步：连接实例

### 9.1 Linux 实例（SSH）

```bash
# 使用密钥对登录
ssh -i ~/.ssh/your-key.pem root@<公网IP>

# 使用密码登录
ssh root@<公网IP>
```

### 9.2 Windows 实例（RDP）

1. 在实例列表点击 **远程连接** → 选择 **VNC 或 RDP**。
2. 下载 RDP 文件，使用本地远程桌面客户端打开。
3. 输入设置的系统密码登录。

### 9.3 GPU 实例初始化检查

```bash
# 验证 NVIDIA 驱动
nvidia-smi

# 验证 CUDA 版本
nvcc --version

# 验证 Docker GPU 支持
docker run --rm --gpus all nvidia/cuda:12.0-base nvidia-smi
```

预期输出：`nvidia-smi` 显示 GPU 型号、显存占用、驱动版本。

## 常见问题

### Q1：创建后能否更换地域？

**不能**。地域在创建时确定，如需迁移请在目标地域新建实例，通过镜像或数据迁移工具搬迁。

### Q2：GPU 实例选错镜像怎么办？

1. 停止实例。
2. 更换系统盘，选择 GPU 镜像。
3. 重新启动，驱动将自动安装。

> **注意**：更换系统盘会清除原有数据，请提前备份。

### Q3：按量付费实例忘记释放怎么办？

建议设置 **自动释放时间**：
1. 实例详情 → 更多 → 设置自动释放。
2. 选择释放时间，到期后系统自动回收实例和按量云盘。

### Q4：创建失败提示"库存不足"怎么办？

- 切换到同一地域的其他可用区。
- 选择其他规格（如从 gn6.large 切换为 gn6.small）。
- 提交工单申请资源预留。

### Q5：如何批量创建多台实例？

1. 在创建向导的「实例数量」处输入目标数量（单次最多 100 台）。
2. 使用实例自定义数据（cloud-init）脚本统一初始化。
3. 也可通过 API / CLI 批量创建：

```bash
smartcloud ecs create \
  --region cn-shanghai \
  --instance-type gpu.gn6.small \
  --image Ubuntu2204-GPU \
  --count 5 \
  --key-name my-key
```
