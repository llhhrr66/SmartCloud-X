# 实例无法登录排查手册

> 文档版本：v1.0
> 最后更新：2026-05-06
> 适用角色：客服、技术支持、运维
> 适用产品：云服务器 ECS、Windows 实例、Linux 实例、远程连接
> 标签：SSH、RDP、登录失败、远程连接、端口、排障

## 快速结论

- 登录不上先看实例是否真的正常运行，再看 IP、端口、安全组、服务和凭证。
- 公网能 ping 通，不代表 SSH 或 RDP 一定正常。
- 安全组放通只是其中一层，系统防火墙和 SSH/RDP 服务状态同样关键。
- 刚做完系统更新、网络变更、驱动升级后失联，通常优先怀疑系统层问题。
- 系统盘打满、服务未启动或启动异常，也会表现为“实例在运行但登不上”。

## 高频问法入口

- SSH 为什么超时
- 3389 为什么连不上
- 控制台显示运行中为什么还登不上
- 公网能通为什么进不去
- 什么时候要转运维深入处理

## 1. 适用范围

本文适用于以下场景：

- Linux 实例 SSH 无法登录
- Windows 实例 RDP 无法连接
- 控制台远程连接失败
- 用户反馈实例能看到，但进不去
- 客服或技术支持需要快速判断问题属于网络、凭证、系统还是资源状态

## 2. 常见故障分类

实例无法登录通常可分为以下几类：

1. 实例状态异常
2. 公网或内网访问路径异常
3. 安全组或防火墙限制
4. 凭证问题
5. 操作系统服务异常
6. 磁盘或系统资源问题
7. 启动异常或系统损坏

## 3. 第一层判断：实例本身是否正常运行

### 3.1 先确认实例状态

应先确认：

- 实例是否处于运行中
- 是否卡在启动中、重启中或异常状态
- 控制台是否提示健康检查异常

### 3.2 是否刚做过高风险变更

例如：

- 更换镜像
- 升级内核
- 修改网络配置
- 安装驱动
- 扩容系统盘后未正确处理
- 磁盘回滚或快照恢复

## 4. 第二层判断：访问路径是否正确

### 4.1 是否使用了正确的 IP

应确认：

- 公网登录时是否使用正确公网 IP
- 内网登录时是否处于正确网络环境
- 是否误用了历史 IP、旧 EIP 或错误 DNS 记录

### 4.2 是否具备公网访问条件

公网连接至少要满足：

- 实例有公网访问能力
- 访问方网络能到达该公网地址
- 没有被中间网络限制

### 4.3 是否走了正确的登录方式

- Linux：通常使用 SSH
- Windows：通常使用 RDP
- 某些场景可使用控制台远程连接或串口、救援方式

## 5. 第三层判断：安全组和防火墙是否放通

### 5.1 Linux SSH 登录失败

先检查：

- 22 端口是否开放
- 来源 IP 是否被允许
- 是否只开放了内网来源
- 系统防火墙是否允许 SSH

### 5.2 Windows RDP 登录失败

先检查：

- 3389 端口是否开放
- 来源 IP 是否被允许
- Windows 防火墙是否允许远程桌面

### 5.3 只检查安全组是不够的

安全组正确后，还要确认：

- 系统防火墙是否允许
- SSH 服务或 RDP 服务是否启动
- 服务监听端口是否被改动

## 6. 第四层判断：凭证是否正确

### 6.1 Linux 常见凭证问题

包括：

- 私钥不匹配
- 密钥权限不正确
- 用户名错误
- 密码错误
- SSH 登录策略限制了 root 或密码登录

### 6.2 Windows 常见凭证问题

包括：

- 密码错误
- 账号被锁定
- 用户名不正确
- 重置密码后未生效或输入错误

### 6.3 凭证问题的识别特点

如果网络可达，但认证后失败，通常更像凭证问题；如果连接阶段就超时或拒绝，通常优先查网络或服务状态。

## 7. 第五层判断：系统服务是否正常

### 7.1 SSH 服务异常

常见表现：

- 端口不监听
- SSH 服务未启动
- 配置文件错误导致服务无法启动
- 系统更新后服务异常

### 7.2 RDP 服务异常

常见表现：

- 远程桌面服务未启动
- 系统策略禁用远程访问
- 系统卡死但未完全关机

### 7.3 系统资源打满导致无法登录

典型情况包括：

- 系统盘满
- 内存极低
- CPU 长期满载
- 日志过大导致系统服务异常

## 8. 第六层判断：系统是否损坏或启动异常

### 8.1 启动异常

如果系统启动过程本身有问题，可能表现为：

- 控制台显示运行中，但实际服务未起来
- 无法通过网络访问
- 串口或系统日志显示启动失败

### 8.2 配置损坏

例如：

- 错误修改网络配置
- 错误修改 SSH 配置
- 文件系统损坏
- 引导配置异常

### 8.3 高风险操作后的连带问题

以下操作后尤其要注意：

- 手工改网卡配置
- 驱动升级
- 内核升级
- 系统盘回滚
- 异常断电或强制关机

## 9. 标准排查流程

### 9.1 第一步：看实例状态

确认实例是否正常运行、有无异常事件。

### 9.2 第二步：看地址和入口

确认 IP、DNS、公网能力、内网路径是否正确。

### 9.3 第三步：看安全组和端口

确认 22 或 3389 是否开放，来源 IP 是否被允许。

### 9.4 第四步：看系统防火墙和服务

确认 SSH 或 RDP 服务是否启动、系统防火墙是否放通。

### 9.5 第五步：看凭证

确认密钥、用户名、密码、登录方式是否正确。

### 9.6 第六步：看系统资源与启动状态

确认磁盘、内存、日志、启动日志、控制台诊断信息是否异常。

## 10. Linux 场景专项说明

### 10.1 SSH 超时

通常优先排查：

- 公网路径
- 22 端口
- 安全组
- 防火墙
- SSH 服务状态

### 10.2 SSH 拒绝连接

通常优先排查：

- 服务未监听
- 端口改了
- SSH 服务未启动
- 本地防火墙拒绝

### 10.3 SSH 提示权限或认证失败

通常优先排查：

- 密钥不匹配
- 用户名错误
- 密钥权限不正确
- 登录方式被禁用

## 11. Windows 场景专项说明

### 11.1 RDP 超时

通常优先排查：

- 公网路径
- 3389 端口
- 安全组
- Windows 防火墙

### 11.2 RDP 账号密码错误

通常优先排查：

- 密码是否已重置
- 用户名是否正确
- 是否用了错误本地账号

### 11.3 远程桌面界面卡住

可能需要排查：

- CPU 或内存不足
- 图形会话异常
- 系统更新中
- 实例负载过高

## 12. 升级处理条件

以下情况建议直接转技术或运维深入处理：

- 安全组和凭证都确认无误仍无法登录
- 控制台诊断提示系统异常
- 刚做完系统级变更后立即无法登录
- 系统盘已满
- 实例存在启动故障或疑似文件系统损坏
- GPU 驱动、网络配置、内核升级后失联

## 13. 工单建议收集信息

建议收集：

- 实例 ID
- 操作系统类型和版本
- 登录方式（SSH、RDP、控制台）
- 使用的 IP
- 报错截图
- 最近是否改过安全组、密码、密钥、镜像、驱动、磁盘
- 问题开始时间
- 是否能通过控制台看到系统日志或诊断信息

## 14. 对外回复建议

### 14.1 “为什么我的服务器突然登不上了”

建议答法：

实例无法登录通常需要分层排查，先确认实例运行状态和访问 IP 是否正确，再检查安全组、系统防火墙、SSH 或远程桌面服务，以及账号密码或密钥是否正确。如果近期做过系统升级、网络变更或磁盘操作，也需要重点检查这些变更是否影响了系统状态。

### 14.2 “我公网 IP 能 ping 通，但 SSH 进不去”

建议答法：

这通常说明网络路径不一定完全中断，但 22 端口是否开放、SSH 服务是否正常、系统防火墙是否允许、密钥或用户名是否正确，仍需进一步确认。

### 14.3 “控制台显示运行中，为什么还连不上”

建议答法：

实例显示运行中只说明虚拟机状态存在，不代表操作系统服务一定完全正常。若 SSH 或 RDP 服务未启动、系统盘打满、系统启动异常或网络配置损坏，仍可能无法登录。

## 17. 标准 FAQ

### Q1：公网 IP 能 ping 通，但 SSH 还是进不去，通常先查什么

A：通常先查 22 端口、安全组、系统防火墙和 SSH 服务状态。能 ping 通不代表 SSH 服务一定正常。

### Q2：控制台显示实例在运行，为什么仍然登不上

A：因为运行中只代表虚拟机状态存在，不代表操作系统服务一定完全正常。系统盘满、启动异常、SSH 或 RDP 服务异常都会导致无法登录。

### Q3：为什么改完安全组还是不行

A：因为安全组只是其中一层。还要继续检查公网地址、系统防火墙、服务监听端口以及账号、密钥或密码是否正确。

### Q4：刚做完系统更新就登不上了，通常说明什么

A：这通常提示问题可能不只是网络，可能与内核更新、驱动变更、SSH 配置错误或启动过程异常有关。

### Q5：什么时候应该直接转技术支持

A：当安全组、IP、凭证都已确认无误，仍然无法登录，或实例存在系统盘满、启动故障、控制台诊断异常时，建议尽快升级处理。
