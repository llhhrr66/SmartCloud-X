# @smartcloud-x/frontend-sdk

Shared frontend baseline for SmartCloud-X web surfaces.

## Scope
- contract-aligned DTO/type exports for auth, orchestrator/chat, admin knowledge flows, marketing, research, and the current owned web-user business outlet for billing / order / ticket / ICP / file / citation-detail
- an owned envelope-aware business contract typing outlet for those web-user business surfaces, including typed `ApiEnvelope` / canonical-success wrappers, named-resource wrappers, offset-page inputs, and explicit endpoint request/response aliases that mirror frozen envelope/pagination conventions while frozen business schemas are still pending
- the owned business contract outlet now also models pagination arriving through shared `ApiEnvelope.meta` / `meta.pagination`, so billing / order / refund / ticket list adapters can stay strict even before those user-business surfaces are promoted into frozen frontend schemas
- the owned business contract outlet now also centralizes surface-specific named resource aliases such as `billing_detail_page`, `order_detail`, `refund_record`, `ticket_detail`, `material_check_result`, `file_upload_policy`, and `citation_detail`, so consumers do not need app-local wrapper detection for those live backend payloads
- shared API client with envelope/error/SSE parsing, a strict shared frontend error-code typing outlet aligned to the frozen `FoundationErrorCode` exports, frozen-foundation error-code classification (including `error_code` / `error_message` aliases, explicit in-band failure-envelope rejection, inferred shared status resolution, nested `error.details` / trace request-id handling, `404` not-found handling, and structured non-OK `text/event-stream` error-body parsing), whitespace-safe CRLF SSE transport parsing, SSE transport error wrapping, reconnect delay helpers that can reuse server retry hints, and a reusable reconnect-consuming helper for browser chat/event streams
- shared request/stream/session error paths now also preserve the caller-generated `X-Request-Id` when backend envelopes omit `request_id`, so telemetry and downstream error handling keep a stable correlation id across HTTP, JSON-envelope SSE failures, and auth/session helper requests
- shared auth/session helpers now also reuse the same transport-error shaping as the core request client, preserving generated request ids and explicit transport status on raw network failures while avoiding session clears for retryable refresh transport errors
- shared structured error-detail extraction for nested `missing_fields`, `required_permissions`, `missing_auth_context`, `missing_payload_hints`, account/confirmation flags, and context-binding maps so chat/session/business consumers can distinguish clarification, permission, and confirmation flows without re-parsing raw envelopes in app code
- shared structured error parsing now also treats frozen `pending_user_actions[]` metadata as actionable continuation state, so retry/reconnect helpers and chat event normalization do not mis-handle server-shaped follow-up-required responses as transient failures
- shared SSE retry/reconnect helpers now also honor explicit frozen `user_action_hint.action` metadata, preventing server-shaped but user-actionable envelopes from being retried as if they were transient transport failures
- page-shaped billing / order / refund / ticket adapters with owned pagination-query/result typings and surface-specific page aliases that reuse frozen pagination conventions while frontend contract promotion is pending
- reusable surface-specific business adapters for `billing`, `orders`, `tickets`, `icp`, `files`, and `citations`, while `serviceDesk` remains the compatibility aggregate for shared workspace composition
- sparse detail adapters now preserve route/fallback identity for refund, ICP, file, and citation-detail reads, preventing `*_unknown` placeholder leakage when the backend omits repeated identifier fields in wrapped responses
- shared order / refund / ticket / ICP business-state helpers for fallback detail shaping, deduped workspace updates, sorted ticket/refund state adoption, and reusable file-upload -> attachment / ICP-material state adapters so pages can stay thin while these surfaces are still owned frontend contracts
- shared load-state helpers now also filter visible failed/fallback domains and resolve the first structured `retryAfterMs` across shared workspace/business load-state errors, so thin page shims do not need to understand the raw load-state envelope shape
- shared load-state builders now also centralize degraded/fallback load-state assembly and ICP page-result shaping, so the live shared adapter and the remaining dev-only mock seam emit the same `IcpApplicationListResult` contract
- raw abort/network transport failures now classify into the same shared timeout/server retry buckets used by structured envelopes, keeping retry decisions consistent across HTTP, auth-session helpers, and SSE-adjacent callers
- shared page-contract helpers now also centralize offset-page slicing/result math for frontend-owned list surfaces, so the live ICP tracked-history fallback path and the app-local mock seam reuse one pagination implementation
- shared degraded/fallback load-state metadata now preserves per-domain structured error info (`kind`, status/code, request id, retry-after) for billing and service-desk surfaces, so consuming apps can distinguish permission failures from conflicts and rate limits instead of only receiving boolean degraded flags
- shared business normalizers for billing page queries and billing/order/ticket/ICP/file mutation inputs so duplicate attachments/materials, malformed page values, whitespace-heavy strings, and malformed billing-cycle hints are cleaned once inside the SDK before idempotency keys or request bodies are built
- shared billing summary query DTOs/normalizers and a strict `billing.getSummary(...)` adapter path, so the billing surface no longer relies on a hand-built summary query string outside the owned contract outlet
- shared owned business enum catalogs/guards for billing summary ranges, ticket priorities/categories, ICP material types, and upload biz types, so thin app shims can reuse one typed frontend outlet instead of re-declaring route-facing lists locally
- shared file/citation mappers plus upload/complete normalizers are also reusable from thin dev-only shims, so mock fallback DTOs can stay aligned to the same owned file / citation contracts as the live adapter path
- shared file/citation fallback builders now package the remaining dev-only mock shaping for upload policies, completed uploads, stored file reads, and citation detail defaults, keeping those explicit degraded paths inside the SDK instead of app-local object literals
- shared business page-query normalizers now accept contract-style `page_size` / `billing_cycle` inputs and numeric-string values, which keeps thin app shims and future contract-facing callers aligned to one owned normalization path
- shared chat/session adapter helpers for the frozen web-user session lifecycle, including session CRUD, retry/cancel, and stream event normalization on top of the shared request client
- shared business normalization that now accepts both wrapped downstream contract payloads and already-normalized shared DTO reuse for composed billing workspace, service workspace, ICP, file, and citation flows, including shared degraded-load state for partial billing/service-workspace failures
- shared ICP history adapters now prefer the live `GET /api/v1/icp/applications` route with owned page typing, persist live-learned ICP application ids for later fallback, expose a strict `IcpApplicationListResult`, preserve requested pagination when they fall back to tracked-detail history, keep successful later live pages backend-shaped instead of locally rehydrating tracked ids into them, and treat transient list failures such as `429 RATE_LIMITED` as degradable shared load-state paths instead of silent empty data
- browser auth/session helpers for user-web integration, including stricter silent-refresh gating for refreshable versus explicitly invalid auth failures and transient refresh handling that preserves the stored session on retryable failures such as `429`
- reusable web-user and web-admin adapters, request-body helpers, a bundled `createWebUserBusinessApis(...)` factory for billing/service-desk/file/citation adoption, admin retrieval mappers, typed admin mutation/query inputs, admin runtime snapshot / capabilities helpers, relative-base-safe admin URL/query handling, and wrapped named-resource normalization for billing/order/ticket/ICP/file/citation detail payloads

## Current adoption
- `apps/web-user` now reuses the shared API client, auth/session manager/store, auth/chat/marketing/research DTOs, billing/order/ticket/ICP/file/citation-detail DTOs and adapters, chat request-body helpers, marketing copy/poster/research/auth request-response helpers, chat/session response mappers, shared billing/service-workspace degraded-state signals, and shared SSE reconnect helpers through thin local shims
- `apps/web-user` business services now also share a single local `createWebUserBusinessApis(...)` instance so billing, order, ticket, ICP, file, and citation adapters reuse one contract-aligned SDK factory instead of re-wiring the same client/idempotency/ICP-tracking setup in multiple service files, and the thin service shim now re-exports the shared page-shaped ICP history result for future page adoption
- `apps/web-user` dev-only file and citation fallback shims now also reuse the shared upload/file normalizers plus file/citation mappers, reducing the last app-local DTO shaping on those owned frontend-business surfaces
- `apps/web-user` service-desk ticket/material selectors now also reuse shared known ticket priority/category and ICP material catalogs/guards, so search-param prefill and form option lists no longer maintain a separate app-local copy of those owned frontend enums
- `apps/web-user` order-center and service-desk pages now also reuse shared order-refund fallback helpers, ticket/refund/ICP workspace update helpers, shared file-upload attachment/material state helpers, and shared ICP list-vs-tracked fallback metadata instead of keeping those assumptions page-local
- `apps/web-user` page shims now also reuse shared load-state selection/retry-after helpers for degraded workspace banners, keeping structured domain-error handling in the SDK instead of page-local array walking
- `apps/web-user` billing page now also consumes the shared load-state selection/retry-after helpers, removing the last local retry-after traversal for degraded billing sections
- `apps/web-user` service-desk mock ICP page shaping now also reuses the shared `buildIcpApplicationListResult(...)` helper, keeping the dev-only fallback aligned with the live shared SDK load-state contract
- `apps/web-admin` now reuses the shared request wrapper, knowledge/rag/admin adapter surface, admin retrieval preview/diagnostics mappers, readiness-aware health DTOs, shared admin mutation/query input types, and shared runtime snapshot / RAG capabilities helpers (including runtime event-counter / outbox detail typings) through a thin local shim

## Known gaps
- billing / order / ticket / ICP / file / citation-detail DTOs are now shared in `packages/frontend-sdk/` through an owned frontend typing outlet, but no frozen shared frontend contract has been promoted for those surfaces yet
- the shared SDK now carries an owned ICP application list page contract because the primary spec still lacks a frozen canonical list contract for `GET /api/v1/icp/applications`; a change request remains the path to frozen promotion
- owned page-query / page-result typings for billing / order / refund / ticket now live in the shared SDK, but the underlying business payloads still need frozen OpenAPI/common-schema promotion before the outlet can collapse onto frozen exports
- the owned business contract outlet is now explicit about canonical/api envelope wrappers and named-resource/page aliases, but those wrappers still map to downstream-owned service payloads rather than frozen user-business schemas
- foundation has now processed [2026-04-16-frontend-sdk-foundation-error-code-export-alignment.md](/home/ljr/SmartCloud-X/docs/contracts/change-requests/2026-04-16-frontend-sdk-foundation-error-code-export-alignment.md), and the frozen exported error catalog now includes `CHAT_STREAM_EVENTS_NOT_FOUND`, so the frontend SDK no longer needs an owned supplemental error-code list for that replay-miss case
- the current orchestrator stream-event replay/list routes still return `CHAT_STREAM_EVENTS_NOT_FOUND` in live service code, but that route-level contract is still absent from frozen `openapi/`; this run continues tracking the route promotion gap through a change request instead of editing frozen contracts directly
- user-facing canonical frontend DTO promotion beyond the currently frozen auth / orchestrator / marketing / research contracts should continue through `docs/contracts/change-requests/`, after which the owned business outlet can collapse onto frozen shared exports
