## 20.2 工程验收

| 条目 | 当前判断 | 备注 |
|---|---|---|
| 支持 Docker 部署 | 已实现 | Compose 文件已覆盖主要服务、前端、基础设施与观测组件 |
| 支持 LangSmith tracing | 部分实现 | `orchestrator-service` 有 LangSmith 配置与 tracing context，但默认依赖 `LANGSMITH_API_KEY` 与环境变量启用 |
| 支持 Phoenix tracing | 部分实现 | `knowledge-service`、`rag-service` 有 OTLP tracing 配置，Compose 中也有 `phoenix` 服务 |
| 支持 Grafana 监控 | 已实现 | Compose 中已包含 `prometheus`、`grafana`、`cadvisor` |
| 支持 Saga 补偿 | 基础版已实现 | `orchestrator-service` 有 compensation stack、rollback 接口与 compensation 执行链路 |
| 支持 Redis 缓存 | 已实现 | 编排、工具查询缓存、幂等与运行态缓存均有 Redis 配置与代码 |
| 支持混合检索 | 部分实现 | `knowledge-service` 具备 Qdrant + OpenSearch 路径，模型中也标注 `hybrid-baseline`，但用户聊天主链路仍未完全收口到真实混合检索结果 |

## 总结

- 这个项目不是“完全没做出来”，而是“很多能力已经有代码落地，但完成度不一致”。
- 最稳妥的结论是：
  - 登录/会话、账单、工单、ICP、后台、Docker、Grafana、Redis：可以认为已经做出来了。
  - 5 个 Agent、营销推荐、Saga：可以认为基础版做出来了。
  - RAG、深度研究报告、LangSmith、Phoenix、混合检索：可以认为做了，但还不能视为完全达到长文档目标态。
